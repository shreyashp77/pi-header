## What We Were Working On
Create a Pi extension that displays custom pi-themed ASCII art at the top of every new session, styled with theme colors, along with model/session/timestamp info.

## What Was Done
- Created extension at `~/.pi/agent/extensions/ascii-header/index.ts`
- Implemented 5 art styles: `classic`, `minimal`, `bold`, `symbol`, `math`
- Registered `/ascii` command with subcommands: `style <name>`, `clear`, `show`
- Header displays on every `session_start` via `ctx.ui.setHeader()`
- Shows model ID, session leaf ID (last 6 chars), session basename, and formatted date/time
- All rendered using `Theme` component from `@earendil-works/pi-tui`

## Key Decisions
- **`setHeader` over `setWidget`**: Chose `ctx.ui.setHeader()` because it renders above chat (not just above editor), matching Claude/Gemini behavior
- **Art styles store raw strings, not themed output**: Styles are theme-agnostic `lines: string[]` arrays; theme is applied at render time inside `AsciiHeader.render()`
- **Store `Theme` object, not `theme.fg` method**: Initial approach captured `theme.fg` as a function reference, which lost `this` binding (JS method extraction issue). Fixed by storing the full `Theme` instance and calling `theme.fg()` with correct `this`
- **Resolve style by name in `render()`**: Style name is captured at `session_start`, but the full `ArtStyle` is looked up inside `AsciiHeader.render()` so it uses the current theme
